HTML

  • Creates the structure of a webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An element describes the entire element from opening to closing tag.
  • A tag is what appears inside the angle brackets.
  • Use header tags in order from h1 to h6, and only use h1 once per page.
  • alt text describes the appearance and functionality of an image. It is important for assistive technology.

CSS

  • Programming language used to define how webpage content is presented to useres. E.g. color, aesthetics, visual layout, organization.
  • Inline CSS stules element directly in the HTML file, applies to specific element only. Least flexible option.
  • Internal style sheet contains CSS rules for the webpage in the 'head' section of the HTML file. Applies to specific page only.
  • External CSS stylesheet keeps rule in a sperate file. Most common method. Sheets linked in 'head' section of the HTML file.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on, and what files are staged
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: addes all changed files to staged files
  • Remember to run 'git pull origin main' before making a new feature branch.
  • git commit -m "commit message here": commits staged changes to current branch.

JavaScript

  • High-level, dynamic programming language that allows users to interact with websites. E.g. effects, interactivity, data communication, etc.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Falsey values are 0, -0, 0n, "", null, unidentified, or false. All other values are truthy.
